# PROJECT_NOTES.md - yun 项目工作约定

这个仓库**只推文档**。

## 红线

- 只允许修改和推送文档文件（例如 `rules.md`、`README.md`）
- 不要把私有代码、测试、引擎实现、脚本推到这个仓库
- 如果发现目录里有源码，也不要提交；先确认是否为私有内容

## 工作方式

1. 先看 `rules.md`
2. 用户给出规则变更时，优先做三件事：
   - 把口语描述整理成规则书风格
   - 检查是否与现有条目冲突
   - 直接改 `rules.md`
3. 改完后提交到 `new` 分支
4. 推送前再次确认：**本次提交只包含文档**

## 文风偏好

- 用规则书语气，短句、清楚、少绕
- 能直接写“防御: 0攻0防”就别写太绕
- 若在定义某个道具本体属性，就不要混入别的机制描述，除非明确标成“补充”或“结算效果”

## 当前已知偏好

- `Magic 星尘` 的“防御”描述应当只说 **Magic 星尘本身的防御**
- `结算` 是单独描述的内容，不要混进“防御”句子里
- 涉及“致使”时，若要改防御，表述成“其 Magic 星尘的防御改为 1”更严谨

## Git

- 目标分支：`new`
- 推送前检查变更文件列表，避免误推非文档内容
