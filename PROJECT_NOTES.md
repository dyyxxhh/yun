# PROJECT_NOTES.md - yun 文档仓库说明

这个仓库是 **文档仓库**。

## 红线

- 只允许修改和推送文档文件（如 `rules.md`、`README.md`）
- 不要提交源码、测试、脚本、服务端实现、判定引擎
- 推送前确认本次提交中 **只有文档**

## 默认工作流

1. 先读 `rules.md`
2. 将用户的口语规则整理成规则书风格
3. 直接修改文档，不要顺手引入私有实现
4. 提交到 `new` 分支
5. 推送前检查改动文件列表

## 文风

- 规则书语气
- 句子短、结论直接
- 定义道具本体属性时，不混入别的机制
- 额外机制请单列为“补充”或“结算效果”

## 当前偏好

- `Magic 星尘` 的“防御”描述只写 **Magic 星尘本身的防御**
- `结算` 单独描述，不要混进“防御”句子
- 若涉及“致使”，更严谨的说法是：**其 Magic 星尘的防御改为 1**
- `结算效果` 可直接写成：**0攻0防**

## Git

- 目标分支：`new`
