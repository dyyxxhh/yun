## 游戏演示

以下是几个游戏场景的演示，帮助你理解游戏规则：

### 演示1：三人游戏，两人攻击一人防御
- **玩家A**：出上勾（攻击=2，防御=0，消耗-2运）
- **玩家B**：出上勾（攻击=2，防御=0，消耗-2运）
- **玩家C**：出防（防御=3，攻击=0，消耗-0运）

**结果**：
- A对C造成2点伤害
- B对C造成2点伤害
- C总共受到4点伤害 > 防御3点
- **C死亡**

### 演示2：三人互相攻击
- **玩家A**：出上勾（攻击=2，防御=0）
- **玩家B**：出上勾（攻击=2，防御=0）
- **玩家C**：出上勾（攻击=2，防御=0）

**结果**：
- A对C造成2点伤害
- B对C造成2点伤害
- C对A造成2点伤害
- C对B造成2点伤害
- 所有伤害相互抵消
- **三人都没有死亡**

### 演示3：两人游戏，特殊道具对决
- **玩家A**：出Magic Fang（魔法獠牙，攻击=1，防御=1，消耗-4魔力）
- **玩家B**：出上勾（攻击=2，防御=0，消耗-2运）

**结果**：
- A对B造成1点伤害
- B对A造成2点伤害
- A有1点防御，抵消1点伤害，实际受到0点伤害
- **两人都没有死亡**

## 贡献指南

欢迎为这个游戏规则文档做出贡献！如果你发现规则有错误、不清晰的地方，或者有改进建议，可以通过以下方式提交PR（Pull Request）：

### 如何提交PR

1. **Fork仓库**
   - 点击右上角的"Fork"按钮，将仓库复制到你的GitHub账户

2. **克隆仓库到本地**
   ```bash
   git clone https://github.com/你的用户名/yun.git
   cd yun
   ```

3. **切换到主分支并更新**
   ```bash
   git checkout main
   git pull origin main
   ```

4. **进行修改**
   - 修改 `rules.md` 文件中的规则内容
   - 确保格式正确，使用标准的markdown语法
   - 保持内容的清晰和准确

5. **提交更改**
   ```bash
   git add .
   git commit -m "描述你的修改内容"
   ```

6. **推送到你的仓库主分支**
   ```bash
   git push origin main
   ```

7. **创建Pull Request**
   - 访问原始仓库：https://github.com/dyyxxhh/yun
   - 点击"Pull requests" → "New pull request"
   - 选择"compare across forks"
   - 选择你的main分支
   - 填写PR标题和描述，说明你的修改内容和原因
   - 点击"Create pull request"

### PR规范

- **标题格式**：`类型: 简要描述`
  - 例如：`fix: 修复道具8的伤害计算错误`
  - 例如：`docs: 添加道具使用示例`
  - 例如：`feat: 新增道具43的规则`

- **类型说明**：
  - `fix`: 修复错误
  - `docs`: 文档改进
  - `feat`: 新增功能/规则
  - `style`: 格式调整（不影响内容）
  - `refactor`: 重构现有内容

- **描述要求**：
  - 清晰说明修改的原因
  - 如果有相关讨论或issue，请引用
  - 描述修改的具体内容
  - 如果有测试或验证，请说明

### 注意事项

1. **保持一致性**：确保修改后的规则与现有规则格式一致
2. **准确性**：规则描述要准确无误，避免歧义
3. **完整性**：如果是新增道具，请提供完整的属性说明
4. **可读性**：使用清晰的markdown格式，方便阅读

感谢你的贡献！🎮
